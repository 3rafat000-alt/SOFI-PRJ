# وثيقة متطلبات المنتج (PRD)
## صكك — SAKK Wallet
### محفظة رقمية متعددة العملات للسوق السوري والشرق الأوسط

---

## 1. المنتج: صكك — محفظة رقمية سورية

**صكك (SAKK)** هي محفظة رقمية متكاملة مصممة خصيصاً للسوق السوري ومنطقة الشرق الأوسط. تهدف إلى توفير حلول دفع حديثة تتجاوز التحديات المصرفية التقليدية في المنطقة، مع دعم كامل للعملتين الرئيسيتين: الدولار الأمريكي (USD) والليرة السورية (SYP).

### الهوية البصرية
- **الألوان**: أبيض رخامي (#F7F3EE) — خمري (#6E1B2D) — ذهبي عتيق (#B58A3C)
- **الخطوط**: IBM Plex Sans Arabic (تطبيق الجوال)، Cairo (لوحة الإدارة)
- **اللغة**: العربية (RTL) كلغة أساسية، الإنجليزية كلغة ثانية

---

## 2. الرؤية والرسالة

### الرؤية
أن نكون المنصة المالية الرقمية الأولى في سوريا والمنطقة، تمكّن الأفراد والشركات من إدارة أموالهم بسهولة وأمان عبر حلول رقمية مبتكرة.

### الرسالة
تقديم خدمات مالية رقمية شاملة وموثوقة تتجاوز الحدود التقليدية، مع التركيز على الأمان والشفافية وسهولة الاستخدام، مع دعم كامل للعملات المحلية والأجنبية لتناسب احتياجات السوق السوري.

### القيم الأساسية
1. **الأمان** — حماية أموال المستخدمين وبياناتهم هي الأولوية القصوى
2. **الشمولية** — خدمات متكاملة تناسب الجميع من أفراد وتجار ووكلاء
3. **الابتكار** — حلول تقنية متطورة تلائم التحديات المحلية
4. **الثقة** — بناء علاقة ثقة مع المستخدمين عبر الشفافية والموثوقية

---

## 3. الشخصيات المستهدفة

### الشخصية 1: الأفراد (Individuals)
- **الوصف**: مستخدمون عاديون يريدون محفظة رقمية لإدارة أموالهم
- **الاحتياجات**:
  - إرسال واستلام الأموال بسهولة
  - دفع الفواتير والشراء عبر الإنترنت
  - تحويل العملات (USD/SYP)
  - ادخار الذهب والاستثمار
  - الحصول على بطاقات رقمية للتسوق
- **نقاط الألم**: صعوبة الوصول للخدمات المصرفية، نقص الخيارات الرقمية، الحاجة للتعامل بالعملتين

### الشخصية 2: التجار (Merchants)
- **الوصف**: متاجر فعلية وإلكترونية تقبل الدفع عبر صكك
- **الاحتياجات**:
  - قبول المدفوعات من العملاء
  - إدارة المبيعات والإيرادات
  - الوصول لواجهات API للتكامل
  - أدوات تقارير وتحليلات
- **المميزات**: واجهة API متكاملة، إدارة المخزون، تقارير المبيعات

### الشخصية 3: الوكلاء (Agents)
- **الوصف**: وكلاء معتمدون لإيداع وسحب النقود (Cash In/Out)
- **الاحتياجات**:
  - استقبال إيداعات المستخدمين نقداً
  - تسليم سحوبات المستخدمين نقداً
  - عمولة على كل معاملة
  - إدارة السيولة المتوفرة
- **المميزات**: ربح عمولات، تصنيف حسب الموقع، إدارة ساعات العمل

## 6. المتطلبات غير الوظيفية

### الأمان
1. **المصادقة**: Sanctum Tokens مع expiry قابلة للتكوين
2. **الترخيص**: AdminMiddleware للمسؤولين، EnsureDeviceCanTransact للمعاملات
3. **التشفير**: جميع كلمات المرور مشفرة بـ bcrypt (12 round)، PIN مشفر
4. **حماية الأجهزة**: كل جهاز جديد يحتاج موافقة + فترة أمان 48 ساعة
5. **2FA**: Google Authenticator مع رموز استرداد (8 رموز)
6. **Rate Limiting**: 60 req/min للـ API، 30 req/min للإدارة
7. **CORS**: مقيد بالنطاقات المسموح بها
8. **Webhook Security**: توقيع HMAC-SHA256 لـ CCPayment، التحقق من التوقيع لـ Stripe
9. **IP Whitelist**: قائمة IP مسموح بها لـ CCPayment webhooks
10. **رقم البطاقة**: مشفر في قاعدة البيانات، مخفي عند العرض

### الأداء
1. **الاستعلامات**: تصفّح paginated (20 عنصر لكل صفحة)
2. **التخزين المؤقت**: Cache للرسوم، الإعدادات، أسعار الصرف
3. **قاعدة البيانات**: SQLite (تطوير)، MySQL/PostgreSQL (إنتاج)
4. **المعاملات**: DB::transaction لضمان Atomicity
5. **قفل الصفوف**: lockForUpdate() لتجنب سباق البيانات في التحويلات

### التوفر
1. **Zaman**: 99.9% uptime (هدف)
2. **النسخ الاحتياطي**: نسخ يومي لقاعدة البيانات
3. **التعافي من الأخطاء**: معالجة الاستثناءات مع تسجيل في logs

### الامتثال
1. **الخصوصية**: سياسة خصوصية وشروط خدمة متاحة
2. **KYC/AML**: توثيق الهوية بـ 3 مستويات
3. **حماية البيانات**: تشفير البيانات الحساسة في قاعدة البيانات

## 8. تدفقات العمل الرئيسية

### تدفق التسجيل
```
مستخدم جديد → تسجيل (بريد/هاتف/باسوورد) → إنشاء محفظة USD افتراضية
→ إرسال رمز تفعيل البريد → تفعيل الحساب → KYC Level 1 (تلقائي)
→ إعداد PIN → بدء الاستخدام
```

### تدفق التحويل P2P
```
مستخدم → اختيار "تحويل" → البحث عن مستخدم (بريد/هاتف/كود إحالة)
→ إدخال المبلغ والعملة → التحقق من حدود KYC → قفل المحفظتين
→ خصم من المحول + إضافة للمستلم → إنشاء معاملتين (صادر/وارد)
→ إشعار المستلم (FCM + in-app) → كاش باك 1% للمحول
```

### تدفق شراء الذهب
```
مستخدم → اختيار "ذهب" → اختيار العيار (24/22/21/18) → إدخال الغرامات
→ عرض السعر + رسوم 1% → التحقق من الرصيد → خصم USD من المحفظة
→ إضافة الغرامات لمحفظة الذهب → تسجيل المعاملة
```

### تدفق الموافقة على الأجهزة
```
جهاز جديد يسجل الدخول → إرسال X-Device-Id → إنشاء جهاز بحالة "معلق"
→ إشعار للأجهزة الموثوقة → المستخدم يوافق من جهاز موثوق
→ تفعيل الجهاز الجديد + فترة أمان 48 ساعة (بدون معاملات)
→ بعد 48 ساعة → الجهاز جاهز للمعاملات
```
