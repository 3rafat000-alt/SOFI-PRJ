## Flow (Mermaid)
```mermaid
flowchart LR
  A[تسجيل الدخول] --> B[لوحة الفندق]
  B --> C{حجز جديد}
  C --> D[بحث عن غرفة متاحة]
  D --> E[تأكيد الحجز]
  E --> F[تسكين النزيل]
  F --> G[إقامة النزيل]
  G --> H[خدمات الغرفة]
  H --> I[إنهاء الإقامة]
  I --> J[إصدار الفاتورة]
  J --> K[غرفة للتنظيف]
  K --> L[غرفة متاحة]
  G -.-> M1{طلب صيانة}
  M1 --> M2[إشعار فريق الصيانة]
  M2 --> M3[إصلاح]
  M3 --> G
  C -.-> C1{لا توجد غرف متاحة}

  C2 --> C
  D -.-> D1{خطأ: تعارض تواريخ}
  D1 --> D2[تواريخ مقترحة بديلة]
  D2 --> D
```

## Ranked Friction Log
1. [High] بطء عملية تسكين النزيل بسبب إدخال البيانات يدوياً
2. [High] صعوبة معرفة حالة الغرفة (نظيفة/تحت التنظيف/تحت الصيانة) في الوقت الفعلي
3. [Med] عدم وجود تكامل مع بوابات الدفع المحلية (مدى، STC Pay)
4. [Med] أخطاء في الفواتير عند تعديل مدة الإقامة
5. [Low] صعوبة إدارة طلبات الصيانة وتتبع حالتها

**Rule:** Every later feature MUST trace to a stage above.
