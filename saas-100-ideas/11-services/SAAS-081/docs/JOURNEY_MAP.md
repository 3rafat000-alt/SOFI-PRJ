# JOURNEY MAP — CemeteryMgt (SAAS-081)
> Owner: Journey Architect · Gate 1 · Persona: أحمد الغامدي

## Flow (Mermaid)
```mermaid
flowchart LR
    A[زائر يبحث عن قبر] --> B[يبحث في التطبيق]
    B --> C[يعرض موقع القبر على الخريطة]
    C --> D[يسجل زيارة عبر QR]
    D --> E[يقيم التجربة]
    C -.-> F[خطأ: موقع غير دقيق]
    F --> G[يبلغ عن خطأ في الموقع]
    G --> H[مدير المقبرة يصحح البيانات]
    H --> C

    I[مدير يسجل دفنة جديدة] --> J[يدخل بيانات المتوفي]
    J --> K[يحدد موقع الدفن]
    K --> L[يطبع شهادة الدفن]
    L --> M[يحدث إشغال المقبرة]
    K -.-> N[خطأ: قطعة غير متاحة]
    N --> O[يعرض قطع بديلة]
    O --> K

    P[عامل صيانة] --> Q[يتلقى تذكرة صيانة]
    Q --> R[ينفذ الصيانة]
    R --> S[يرفع تقرير بالصور]
    S --> T[مدير يعتمد]
```

## Ranked Friction Log
1. [High] دقة GPS في المقابر القديمة ذات التخطيط غير المنتظم
2. [High] صعوبة قراءة سجلات الدفن الورقية القديمة (بخط اليد)
3. [Med] الزوار لا يعرفون استخدام QR code
4. [Med] حساسية أسر المتوفين عند التعامل مع البيانات
5. [Low] تكامل نظام إصدار الشهادات مع الأنظمة البلدية

**Rule:** Every later feature MUST trace to a stage above.
